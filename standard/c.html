CSS书写规范

为何规范？

  • 1、规范不是一种约束，而是一种约定，强调团队的一致性
  • 2、加强团队的合作性，提高协作效率
  • 3、形成一种团队文化，积累知识
  • 4、规范最终是为项目服务的，我们所做的一切都是为了优化项目和流程提高我们的工作效率

1、使用CSS缩写属性

CSS有些属性是可以缩写的，比如padding,margin,font等等，这样精简代码同时又能提高用户的阅读体验。
例如:
.css {
padding-top: 10px;
padding-bottom: 1px;
padding-right: 12px;
padding-left: 10px;
}
我们可以缩写
.css {
padding:10px 12px 1px 10px;
}

2、去掉小数点前的“0”

font-size: 0.8rem;我们可以去掉0：font-size:.8rem;

3、CSS选择器命名规范

一、命名禁止用汉字拼音，统一用英文，不会英文百度。
二、简写类名，但前提是要让人看懂你的命名才能简写哦！例如：.navigation我们可以简写为nav，
但.author就不能简写为.aut,因为根本不知道你写的什么!
三、长名称或词组可以使用中横线来为选择器命名。不建议使用“_”下划线来命名CSS选择器，为什么呢？
1、输入的时候少按一个shift键；
2、浏览器兼容问题 （比如使用_tips的选择器命名，在IE6是无效的）

4、16进制颜色代码缩写

color:#fffff;我们可以缩写为color:#fff;

5、不要随意使用Id

id在JS是唯一的，不能多次使用，而使用class类选择器却可以重复使用，另外id的优先级优先与class，所以id应该
按需使用，而不能滥用。

6、为选择器添加状态前缀

有时候可以给选择器添加一个表示状态的前缀，让语义更明了，比如下面是添加了“.is-”前缀。
.withdrawal{}我们添加.is-withdrawal{}

7、书写注释，方便程序开发嵌套。注释格式，统一使用双斜杠加*。：

开始注释：/* 注释 start */
结束注释：/* 注释文案 end */
允许只有开始注释

8、勿使用冗余低效的 CSS 写法，例如：ul li a span{... }

9、慎用 !important

10、开发过程中的未定事项，须用 [!] 标出，以便于后续讨论整理。

11、上下模块之间的间距统一使用下一个模块的margin-top来实现，好处是：如果没有下一个模块也不会多出一段空隙。

12、hover，selected，disabled，current等具有特定意义的请勿直接占用。

13、:link :visited :hover :active书写顺序 L-V-H-A，速记：LoVe（喜欢）HAte（讨厌）。

14、不要使用 @import

15、为JavaScript预留钩子的命名,请以J_起始,比如:J_hide,J_show;用于调用
